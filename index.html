Protocol Flow

How SkillSync Works

Every step maps to a real networking concept — from DHCP-style registration to TCP connection establishment.

01

Register Your Node

Sign up and list your skills and availability. The server registers you like a DHCP client getting an IP address — your username is your logical network address.

02

Match Detection (Graph Routing)

The server runs a compatibility algorithm across all connected nodes — like Dijkstra computing shortest paths. Bidirectional skill matches create edges in the skill graph.

03

3-Way Handshake

Send a connection request (SYN). Your match accepts (SYN-ACK). The server opens a private channel (ACK). The session is now ESTABLISHED.

04

Real-Time Exchange

Messages flow over WebSocket multicast within your private room. Typing indicators use a debounced signal — like TCP's Nagle algorithm.

05

Rate & Build Reputation

Rate sessions to earn points. Reputation updates your routing metric, improving your rank in future matches — like updating OSPF link costs.

Computer Networks Concepts

Networking Principles in Action

SkillSync is a complete demonstration of CN concepts from the textbook — implemented in a real working application.

🔄

TCP/IP Model

HTTP/1.1 carries SPA shell (Application). WebSocket frames carry messages (Transport). All over TCP (Network + Data Link).

📡

WebSocket Protocol

Full-duplex over a single TCP connection. HTTP Upgrade → 101 Switching Protocols. Eliminates polling — push latency O(1).

🗺️

Routing Algorithms

Compatibility score = composite OSPF-style metric. Match detection = Dijkstra on the skill graph. ARP table = sid_map dictionary.

🔀

Multiplexing

Many logical chat channels over one WebSocket connection. Socket.IO rooms handle the mux/demux layer transparently.

🛡️

Loop Prevention

Messages are delivered with skip_sid — the sender's socket is excluded from the multicast (split-horizon / poison-reverse).

💀

Fault Detection

Disconnection triggers reverse ARP lookup and LSA broadcast — like OSPF detecting a failed link and flooding topology updates.

Help Center

Everything You Need to Know

Quick answers to the most common questions about SkillSync.

🔐 How do I create an account?

Click Sign Up, enter your email, choose a username and password. Your account is stored in your browser's localStorage and ready instantly.

🤝 How does matching work?

Bidirectional matching: you must offer what they want AND they must offer what you want. A weighted compatibility score (skills 50% + availability 30% + reputation 20%) is shown as a % ring on each card.

💬 How does chat work?

Once a connection request is accepted, a private Socket.IO room opens. Messages are delivered in real time with typing indicators. Send a message by pressing Enter or the Send button.

⭐ What is reputation?

Reputation is earned by accepting connections, completing sessions, and receiving high ratings. More reputation improves your rank in match results and unlocks badges.

🌐 What CN concepts are demonstrated?

TCP/IP stack, WebSocket protocol (RFC 6455), 3-way handshake, Socket.IO rooms (multicast), broadcast/unicast/multicast delivery, OSPF-style routing metrics, ARP table analogy, split-horizon loop prevention, and fault detection via LSA flooding.
